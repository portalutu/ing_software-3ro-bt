# Kit Completo de Documentación de Proyecto (adaptado para el curso)

Guía para estudiantes de la materia **Ingeniería de Software**, UTU EMT.

Este documento contiene un compendio de los **principales artefactos de documentación utilizados en proyectos de software reales**. Cada sección contiene:

1. Template vacío para ser utilizado en la documentación del proyecto final.
2. Ejemplo concreto de uso para ser usado como "ayuda-memoria" a la hora de llenar el documento real.

## Ejemplo

**Nombre del proyecto:** Sistema de Gestión de Biblioteca Escolar

**Patrocinador:** Director del liceo

**Director del proyecto:** Docente de Ingeniería de Software

**Equipo:** 4 estudiantes del grupo "Los mejores de todos", 3º Año EMT 2026

**Fecha de inicio:** 10/04/2026

**Objetivo del proyecto:**
Desarrollar una aplicación web que permita registrar libros y gestionar préstamos.

**Justificación:**
Actualmente el control de préstamos se realiza en papel, generando errores y pérdida de información.

**Alcance inicial:**

- Registro de libros
- Registro de usuarios
- Gestión de préstamos
- Reportes básicos

**Stakeholders:**

- Dirección
- Bibliotecario
- Estudiantes

**Riesgos:**

- Falta de experiencia técnica
- Falta de tiempo

**Criterios de éxito:**

El sistema debe registrar préstamos correctamente y generar reportes.

# Estructura recomendada del repositorio del proyecto

```
project/

01_project_charter.md
02_stakeholders.md
03_scope.md
04_wbs.md
05_schedule.md
06_risk_register.md
07_communication_plan.md
08_requirements.md
09_backlog.md
10_sprint_plan.md
11_architecture.md
12_test_plan.md
13_final_report.md
```

Esta estructura permite mantener la **documentación organizada como en proyectos reales de ingeniería de software**.
